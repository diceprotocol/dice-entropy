# Versioning Policy

Dice Protocol public packages use semantic versioning.

- **MAJOR**: Breaking SDK API changes, contract interface changes, or behavior changes requiring integrator action.
- **MINOR**: Backward-compatible SDK features, additional examples, new helper methods, and documentation expansions.
- **PATCH**: Bug fixes, typo fixes, packaging fixes, and internal changes that preserve public behavior.

The deployed DiceEntropy contract is immutable. Contract-level changes require a new deployment and explicit migration documentation.
